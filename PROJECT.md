# WordMaster — 从背单词工具到全栈项目

## 一句话定位

**给自己写一个愿意每天打开的背单词软件。**

市面上所有背单词 App 都在想方设法让你上瘾，但没有一个是你自己写的。这个项目从单文件 HTML 起步，一步一步长成有云端、有账户、有排行榜的完整产品——每一步都对应你技术栈的一次升级。

---

## 当前状态（v0）

| 维度 | 现状 |
|------|------|
| 形态 | 单文件 HTML，浏览器直接打开 |
| 存储 | localStorage（浏览器本地） |
| 词库 | 内置 ~150 CET-4 + ~50 CET-6，支持手动导入 |
| 算法 | 简单的 5 级递进复习（5-30-240-1440-10080 分钟） |
| 数据 | 每日学习量、掌握数、待复习数、学习时长 |

**v0 要解决的核心问题：** 先把背单词这个动作跑起来，验证"自己做的工具自己会用"这个前提。

## 成长路线图

### v1：打磨单机体验

> **技术栈：** 纯前端，HTML + CSS + JavaScript  
> **你的能力匹配度：** 不需要写，用 AI vibe coding 就能做  
> **学习目标：** 顺便熟悉 JS 的 DOM 操作、事件、定时器、localStorage

| 功能 | 说明 |
|------|------|
| 艾宾浩斯复习 | 把当前的简单 5 级换成正经的艾宾浩斯曲线（1, 2, 4, 7, 15, 30 天） |
| 多模式 | 闪卡模式 / 选择题模式 / 拼写模式 |
| 词库管理 | 浏览词库、查看每个词的掌握状态、手动增删 |
| 学习日历 | 类似 GitHub 贡献热力图，展示每日学习量 |
| 数据导出 | 导出/导入 JSON，方便备份 |
| 音标发音 | 接 Web Speech API，点击朗读单词 |

### v2：引入后端

> **技术栈：** 前端 + 后端 API + 数据库  
> **你的能力匹配度：** C++ 可以做后端，但更推荐借这个项目学 Go 或 Node.js  
> **学习目标：** 理解客户端-服务器模型、REST API、数据库 CRUD

| 功能 | 说明 |
|------|------|
| 用户注册登录 | JWT 鉴权，最基础的用户系统 |
| 云端同步 | 学习进度存服务端，换设备不丢数据 |
| 单词云端词库 | 词库从数据库读取，不再硬编码在 HTML 里 |
| 简单管理后台 | 看得到自己的学习数据统计 |

**这里有个技术选型的分叉口：**

- **用 Go：** 大厂后端主流，语法简单，并发原生支持，对 C/C++ 背景很友好
- **用 Node.js：** 前后端都是 JS，学习成本低，生态最丰富
- **用 C++：** 你最熟，但做 Web 后端不主流（除非你就是要练 C++）

我的建议：v2 阶段可以先用你熟悉的，甚至用 C++ + httplib 跑起来也行。等 v3 有性能需求了再切 Go。

### v3：社交化

> **技术栈：** 前面基础上 + WebSocket + Redis + 可能的消息队列  
> **你的能力匹配度：** 这时候你应该大二了，数据结构学完、计网学完，正好练手  
> **学习目标：** 实时通信、缓存策略、排行榜算法

| 功能 | 说明 |
|------|------|
| 好友系统 | 关注/取关、好友列表 |
| 排行榜 | 周榜/月榜/总榜，按学习单词数排名 |
| 学习动态 | 好友今天学了几个词，掌握了几个 |
| 单词对战 | 实时 1v1，随机出词比谁先答对 |
| 成就系统 | "连续打卡30天"、"词汇量破1000"之类的徽章 |

排行榜是个经典后端面试场景（千万用户排行榜怎么做？Redis 有序集合 → 分段排名 → 最终一致性），做完这个你能聊半小时。

### v4：真正的产品化

> **技术栈：** 前面基础上 + 移动端 + CI/CD + 监控  
> **你的能力匹配度：** 这时候你应该在大二下学期或大三，该准备实习了  
> **学习目标：** 完整的产品交付链路

| 功能 | 说明 |
|------|------|
| 移动端 | React Native / Flutter，或者直接 PWA 先顶上 |
| 离线支持 | Service Worker，没网也能背 |
| 智能推荐 | 根据错误率调整出词策略（这个可以很浅也可以很深） |
| 性能优化 | CDN、懒加载、接口合并、数据库索引优化 |
| 监控告警 | 接口延迟、错误率、用户活跃度 dashboard |

  ## 为什么这件事值得做？

1. **这是你自己的刚需。** 你要考四六级，每天就要背单词。自己做的工具你更愿意打开——这叫"狗食效应"（dogfooding）。

2. **这是一份会成长的作品集。** 大二找实习的时候，你简历上写"独立开发了一个有 XX 用户的背单词平台"比"在课程设计里实现了一个学生管理系统"强一百倍。面试官能看到你在一个项目上持续投入了两年，经历了从单机到云端到社交化的完整演进——这本身就是最好的工程能力证明。
 
 
3. **每个阶段都对应你课内知识的实战化。**
   - 数据结构学完 → 优化词库的存储和检索
   - 计网学完 → 设计 API 协议、理解 HTTP 缓存
   - 数据库学完 → 设计表结构、建索引、写慢查询
   - 操作系统学完 → 理解并发、锁、连接池
   - 做这个项目的过程就是把课内知识"翻译"成工程实践的过程

4. **后端方向的岗位，最看重的是"你维护过一个真实项目"。** 课设和算法题只能证明你能学，但持续迭代一个项目证明你能做工程。

## 协作方式

- **v0 阶段：** vibe coding 为主，我写代码你验收，重点是把产品方向定下来
- **v1-v2 阶段：** 逐步过渡到你写核心逻辑 + AI 辅助，每个新功能你先想怎么做，再让我实现
- **v3 往后：** 你自己主导架构设计，AI 做代码加速

这个文件就是项目的"宪法"。以后每次想加新功能，先回来看这个路线图——现在的决定应该为下一个阶段铺路，而不是添堵。
