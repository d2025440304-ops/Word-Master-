继续优化 WordMaster 的交互体验，本次只改 UX，不加新功能，不动算法和数据结构。

## 问题 1：会话结束后没有出路（最高优先级）

当前：15题做完进入 empty state，只有一行文字，用户不知道能否继续。
修改：empty state 加一个"继续学习"按钮，点击调用 buildSession() + renderCard()。
同时：buildSession 里如果捞不到15个词，就捞能捞到的，哪怕只有1个也继续，不要直接给空状态。

## 问题 2：答错后改为手动推进

当前：选择题答完800ms自动跳，拼写答完1000ms自动跳，答错了来不及看正确答案。
修改规则：
- 答对 → 保持自动跳（时间可适当延长到1200ms）
- 答错 → 停在当前词，展示完整信息，出现"知道了 →"按钮，用户点击才推进
- 拼写答错时：除了显示正确单词，还要补上音标和中文释义（现在只显示英文）
- 选择题答错时：正确答案高亮后不要立即跳，等用户点"知道了 →"

## 问题 3：工具栏按钮加文字

当前：4个圆形 emoji 按钮，移动端猜不出功能。
修改：每个按钮图标下面加一行文字标签，布局从圆形变成"图标+文字"的小方块。
四个按钮文案：日历 / 导入 / 重置 / 词库（词库按钮同时显示当前词库名，如"CET-4"）。

## 问题 4：拼写答对后展示完整信息

当前：答对只弹"✅ 正确！"然后1秒消失，没有展示音标和释义。
修改：答对后在结果区展示"✅ 正确！[音标] 释义"，让用户看完再自动跳（延长到1500ms）。

## 问题 5："剩余新词"改为今日进度

当前：显示整个词库里 level=0 的词数量，初始就是150+，没有激励感。
修改：把这格改为"今日进度"，显示 今日已学 / 每日目标（DEFAULT_DAILY_GOAL），
格式："12/30"，进度条满了显示"✅"。

## 约束
- 不改算法，不改数据结构，不改 localStorage 格式
- 每个问题独立修改，改完一个告诉我，我确认后再改下一个
- 不要顺手加其他功能



# 第二次更新
对 WordMaster 做一次视觉重设计，目标风格参考 Apple 系统应用和 X（Twitter）
——大字、强层级、克制配色、响应式布局。不改任何 JS 逻辑，只动 CSS 和 HTML 结构。

## 1. 整体布局

桌面端（>768px）：
- 左栏固定宽度 280px：放统计栏 + 工具栏，垂直排列
- 右侧主区域：放模式切换 + 卡片 + 操作按钮
- 左右之间有一条细分割线，无阴影无圆角，干净

移动端（≤768px）：
- 恢复单列，统计栏在上，卡片在中，工具栏在底部固定

## 2. 统计栏重设计

取消8格等权重的 grid，改为明确层级：

第一级（最大）：今日已学——字号 48px，粗体，深色，占据明显空间
第二级（中等）：已掌握 / 正确率 / 学习时长——三个并排，字号 20px
第三级（最小）：连续打卡 / 总词库 / 今日进度——字号 13px，用 text-secondary 颜色，一行排完

去掉每个格子的白色卡片背景和圆角，改为直接显示在左栏背景上，减少视觉噪音。

## 3. 卡片重设计

卡片是页面绝对主角：
- 高度从 280px 增加到占据剩余主要空间，至少 360px
- 单词字号从 36px 增大到 52px
- 音标字号从 15px 增大到 18px，颜色更浅作为次要信息
- 卡片背景纯白，阴影更深更扩散（0 8px 40px rgba(0,0,0,.12)）
- "点击卡片查看释义"提示改为更克制的样式，字更小更浅

## 4. 配色收敛

当前问题：紫色 primary、绿色 success、红色 danger、黄色 unsure 全部高饱和度同时出现。
修改：
- 强调色只保留一个：--primary: #000000（黑色，苹果风格）或保留现有紫色但降低其他颜色饱和度
- 成功绿降饱和：#34c759（苹果系统绿）
- 危险红降饱和：#ff3b30（苹果系统红）
- 所有 box-shadow 的颜色都改为 rgba(0,0,0,x)，去掉彩色阴影

## 5. 按钮重设计

- "掌握"按钮：纯黑背景 + 白字（最重要操作，最重）
- "不会"按钮：白底 + 黑色边框 + 黑字（次要）
- "模糊"按钮：浅灰背景 + 深灰字（最次要）
- 三个按钮圆角统一为 12px，去掉彩色 box-shadow

## 6. 字体层级

- 全局 font-size: 15px（现在是默认16px，苹果用15px更精致）
- 减少 bold 的使用频率，只有最重要的数字和单词用 700，其余用 500 或 400

## 约束
- 不改任何 JS 逻辑和数据
- 媒体查询断点只用 768px 一个
- 不引入外部字体或图标库
- 桌面和移动端分别截图给我看，改完一步我确认再继续
